## 2. 数据库组织规范

### 2.1 数据库命名

每个市场对应一个库，具体库名待定。

### 2.2 表命名规范

表名统一采用 `{账户编码}_{策略编码}_{表名}` 格式。

### 2.3 字段规范

各表中的 `strategy_id` 字段仍然保留，用于记录策略的唯一标识（与表前缀编码可不一致，支持更灵活的策略命名体系）。

### 2.4 建表示例

```sql
CREATE DATABASE ly;
USE ly;

CREATE TABLE ly_maop_operation (
    id               BIGINT          PRIMARY KEY,
    market_id        TINYINT         NOT NULL COMMENT '1-沪深 2-港股 3-美股 4-期货 5-加密货币',
    account_id       VARCHAR(64)     NOT NULL,
    strategy_id      VARCHAR(64)     NOT NULL,
    symbol_code      VARCHAR(32)     NOT NULL,
    -- ...其余字段同"1.1 操作表"定义
);
```

一套基础表结构（operation、deal、asset、trade、position）对应一个策略；每接入一个新策略，按上述命名规范创建一套对应的表即可。
